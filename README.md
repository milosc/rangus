# Rangus

## Discovery Phase

This project is structured for the HTEC Discovery workflow.

### Folder Structure

- **Client_Materials/**: Source materials from client
  - PDFs/: PDF documents
  - Interviews/: Interview transcripts and notes
  - Screenshots/: UI mockups and screenshots
  - UserManuals/: Existing documentation
  - Videos/: Video content
  - Spreadsheets/: Data files and spreadsheets
  - Documents/: Other documents

- **_state/**: Discovery progress and configuration (auto-generated)
- **traceability/**: ID registries and traceability chains (auto-generated)
- **ClientAnalysis_Rangus/**: Discovery outputs (auto-generated)

### Getting Started

1. Upload client materials to appropriate subfolders in `Client_Materials/`
2. Run Discovery analysis: `/discovery Rangus Client_Materials`
3. Review outputs in `ClientAnalysis_Rangus/`

### Notes

- Auto-generated folders (_state, traceability) are created during Discovery execution
- Do not manually edit auto-generated files
- All source materials should be placed in Client_Materials subfolders
